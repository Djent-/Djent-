Hangups With bspwm Config

June 24 2016

To go along with my desktop theming, I wanted to mess with borders in bspwm. This lead me through the process of actually updating my .config/bspwm/bspwmrc file to 0.9.1. I had previously been making config modifications through a startup.sh file I would run on startup, but that is not very clean. To begin, I removed everything that was left from the default bspwmrc that I started with - particularly the bspc monitor -d I II III IV... which I had abandoned for just three virtual desktops - Desktop1, Desktop2, and Desktop3. In bspwm 0.9.1, this needs to go after all bspc config commands. I replaced this with three calls for bspc monitor -d DesktopX after adding the desktops with bspc monitor -a DesktopX.

Before I added the bspc monitor -d commands, I was wondering why my window border changes were not taking affect. That was why - bspc monitor -d refreshes the settings of the monitor.

One last thing that tripped me up, and is the most embarrasing is the border colors. I was trying to set them with bspc config focused_border_color #FFFFFF but only after an hour of searching did I realize that I needed to escape the #!

Anyways, my updated bspwmrc file will be in my arch-config-2015 repository - in the 2016 branch of course! Not sure what I'll do with all my themeed config files. It doesn't make much sense to put them all in my config repository - they might need their own themes-dotfiles repository or something.
